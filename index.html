Для чого проводився аудит
🎯 Мета аудиту
Перевірити чи захищена мережа клініки від несанкціонованого доступу. Знайти пристрої з відкритими небезпечними портами. Оцінити ризик для медичного обладнання та даних пацієнтів.
⚠️ Що виявлено
Медичні лазери, VISIA-станція, камери і сигналізація знаходяться в одній незахищеній мережі без розділення. Будь-хто підключений до Wi-Fi клініки має теоретичний доступ до всього обладнання.
Яким інструментом проводилось сканування
🔍
Nmap — інструмент сканування мережі
▼
⚙ Технічно
Nmap (Network Mapper) — утиліта з відкритим кодом для дослідження мереж. Надсилає TCP/UDP/ICMP-пакети і аналізує відповіді. Визначає: активні хости, відкриті порти, версії сервісів, ОС. Три фази: ping sweep → port scan → service detection (-A).
💼 Для власниці
Nmap — це як цифровий рентген мережі. Програма «стукає» в кожні двері кожного пристрою і записує: хто відповів, яка кімната, яка замок. За 5 хвилин отримуємо повну карту всього що підключено до Wi-Fi клініки.

Що таке порт і чому це важливо
🚪
Мережевий порт — що це і чому відкритий порт небезпечний
▼
⚙ Технічно
Порт (0–65535) — логічна точка завершення мережевого з'єднання. Кожен сервіс слухає свій порт. Стани: open (відповідає), closed (порт відхиляє), filtered (фаєрвол блокує). Порти 0–1023 — системні, вище — динамічні.
💼 Для власниці
IP-адреса = адреса будинку. Порт = номер квартири. Порт 80 = головні двері (вебсайт). Порт 445 = комора з файлами. Порт 554 = вікно з відеокамерою. Відкритий непотрібний порт — це незачинені двері в кімнату де стоїть лазер за $150,000.
Небезпечні порти знайдені в клініці:
21FTP — передача файлів без шифруваннякритичний
23Telnet — термінал без шифрування, пароль виднокритичний
445SMB — файловий обмін Windows, EternalBlueкритичний
554RTSP — відеопотік камер, часто без паролявисокий
9090XMLRPC — API медапарату, відповідає без авторизаціїкритичний
Що таке сегментація та ізоляція мережі
🏗️
Flat network vs. VLAN — різниця між захищеною і незахищеною мережею
▼
⚙ Технічно
Flat network — всі пристрої в одному broadcast-домені L2. Будь-який пристрій може ініціювати з'єднання до будь-якого іншого.

VLAN — Virtual LAN, логічна ізоляція на рівні L2. Різні VLAN не можуть спілкуватись без явного дозволу роутера (inter-VLAN routing + ACL). MikroTik підтримує нативно.
💼 Для власниці
Зараз: клініка — це один великий зал без стін. Гість з телефоном, лазер за €200k і касовий апарат — всі в одній «кімнаті», бачать одне одного.

Після VLAN: між кімнатами з'являться стіни. Гість може дивитись в інтернет але не може «зайти» до лазера. Охоронець перевіряє кожного хто хоче пройти між кімнатами.

🎭 Критичний вектор — Соціальна інженерія підтверджена
⚙ Технічно
Вектор: pretexting. Аудитор представився і отримав пароль від внутрішньої мережі без жодної верифікації особи, протоколу або підпису. Пароль надав адміністратор на усне прохання. Час від першого контакту до підключення до мережі — ~4 хвилини.
💼 Для власниці
Будь-хто може зайти до клініки, представитись «IT-спеціалістом від виробника лазера» і отримати пароль від мережі де стоять ваші апарати. Жодна техніка не потрібна. Це найдешевша атака і найдорожчий провал захисту.

PCI DSS — правильне підключення POS-термінала
❌ Як є зараз (порушення PCI DSS)
PAX (.21) в тій самій мережі що лазери, камери і ноутбуки персоналу. Дані карток пацієнтів потенційно видимі через ARP poisoning. Банк може заблокувати еквайринг при аудиті PCI DSS.
✅ Як має бути (PCI DSS VLAN)
Окремий PCI VLAN тільки для термінала. Firewall: дозволити вихідний трафік виключно на IP еквайринг-банку. Заборонити будь-який доступ до PCI VLAN ззовні.
# MikroTik: ізоляція POS-термінала
/interface vlan add name=pci-pos vlan-id=50 interface=bridge
/ip address add address=10.50.0.1/24 interface=pci-pos
/ip firewall filter add chain=forward in-interface=pci-pos dst-address=[BANK_IP] action=accept
/ip firewall filter add chain=forward in-interface=pci-pos action=drop # все інше заборонено

Критичний
▼
CVE MS17-010 — EternalBlue / WannaCry
Що це: критична вразливість у протоколі SMBv1 (файловий обмін Windows). Дозволяє виконати довільний код на вразливому комп'ютері без будь-якого пароля. Саме цією вразливістю скористався вірус WannaCry у 2017 році, який зупинив сотні лікарень по всьому світу.

1

Словник термінів — технічні та бізнес-пояснення
🌐
IP-адреса
▼
⚙ Технічно
Унікальний числовий ідентифікатор пристрою у мережі. Формат IPv4: чотири числа від 0 до 255 через крапку. Приватні діапазони: 192.168.x.x, 10.x.x.x. У клініці: підмережа 192.168.50.0/24 — до 254 пристроїв.
💼 Для власниці
Адреса будинку у цифровому місті. Кожен пристрій підключений до мережі має свій унікальний «номер будинку». За ним можна знайти пристрій і «постукати у двері».
🚪
Порт (мережевий)
▼
⚙ Технічно
Числовий ідентифікатор (0–65535) що визначає конкретний сервіс на пристрої. Порт 80 = HTTP, 443 = HTTPS, 22 = SSH, 445 = SMB, 554 = RTSP. Firewall може відкривати або блокувати окремі порти.
💼 Для власниці
Номер квартири у будинку. IP = адреса будинку, порт = номер квартири. Відкрити «не ту квартиру» — це залишити вхід для зловмисника до конкретного сервісу (файли, відео, управління).
🏗️
VLAN (Virtual LAN)
▼
⚙ Технічно
Логічне розділення мережі на ізольовані сегменти на рівні L2 без фізичного розмежування. Пристрої різних VLAN не можуть спілкуватись без inter-VLAN routing з ACL. Налаштовується на керованих комутаторах і роутерах (MikroTik підтримує нативно).
💼 Для власниці
Невидимі стіни у мережі. Гість може користуватись інтернетом але не може «зайти» в кімнату де стоїть лазер. Лазер може оновлюватись від виробника але не може «бачити» касовий апарат. Все на одному обладнанні — але ізольовано.
🦠
SMB (Server Message Block)
▼
⚙ Технічно
Протокол мережевого файлового обміну Windows. Порт 445. SMBv1 містить вразливість MS17-010 (EternalBlue) — Remote Code Execution без авторизації. WannaCry використовував саме цю вразливість. «SMB2 negotiation failed» у nmap = SMBv1 активний.
💼 Для власниці
Система обміну файлами між Windows-комп'ютерами. Стара версія (SMBv1) має «дірку» — зловмисник може зайти в комп'ютер без пароля. Саме через неї поширився вірус WannaCry у 2017 що зупинив NHS (британська охорона здоров'я).
📹
RTSP (Real Time Streaming Protocol)
▼
⚙ Технічно
Протокол передачі відео в реальному часі. Порт 554. URL формат: rtsp://[IP]:554/h264/ch1/main. Без автентифікації або з default credentials — відеопотік доступний будь-кому у мережі через VLC або інший медіаплеєр.
💼 Для власниці
«Пряма трансляція» з камер. Якщо камера без пароля або зі стандартним паролем — будь-хто підключений до Wi-Fi клініки може відкрити у звичайному відеоплеєрі і дивитись що відбувається у процедурних кабінетах. В реальному часі.
📋
CVE (Common Vulnerabilities and Exposures)
▼
⚙ Технічно
Міжнародна база даних публічно відомих вразливостей у ПЗ та обладнанні. Формат: CVE-[РІК]-[НОМЕР]. CVSS-оцінка від 0 до 10 описує критичність. 9.0+ = критична. CVE публічно доступні разом з описом і часто proof-of-concept exploit-кодом.
💼 Для власниці
«Реєстр відомих дірок» у різних пристроях і програмах. Як тільки публікується CVE — хакери по всьому світу знають як використати цю «дірку». Якщо пристрій не оновлений — він вразливий до всіх відомих CVE. У вашій клініці знайдено кілька пристроїв з критичними CVE.
🎭
Соціальна інженерія / Pretexting
▼
⚙ Технічно
Клас атак що маніпулюють людьми а не технологіями. Pretexting — створення вигаданого сценарію для отримання інформації. Vishing — через телефон. Phishing — через email. У цьому аудиті: аудитор отримав пароль від мережі через просте усне прохання без верифікації.
💼 Для власниці
Найдорожча атака, яку неможливо заблокувати файєрволом. Зловмисник телефонує і каже «я від Candela, потрібен доступ до Wi-Fi для оновлення лазера». Адміністратор дає пароль. Готово — зловмисник у вашій мережі. Захист: навчання персоналу і чіткий протокол.
🔐
GDPR та медичні дані
▼
⚙ Технічно
GDPR (General Data Protection Regulation) — регламент ЄС щодо захисту персональних даних. Медичні дані = Категорія 9 (спеціальна категорія). Максимальний штраф: €20,000,000 або 4% річного глобального обороту. VISIA зберігає фото обличчя = біометричні дані = найвищий рівень захисту.
💼 Для власниці
Якщо дані пацієнтів (фото з VISIA, відео з камер) потраплять до сторонніх — штраф до €20 мільйонів. Для клініки з іноземними пацієнтами це особливо актуально. Незахищена мережа = порушення GDPR. Захищена = compliance і конкурентна перевага.
🔁
Ransomware
▼
⚙ Технічно
Тип шкідливого ПЗ що шифрує файли жертви і вимагає викуп за ключ розшифрування. Розповсюджується через SMB (WannaCry/NotPetya), фішинг або RDP. У пласкій мережі без VLAN один заражений хост може зашифрувати всі пристрої за хвилини.
💼 Для власниці
Вірус що «замикає» всі комп'ютери і вимагає гроші за «ключ». Якщо один ноутбук адміністратора заражений — без VLAN за 10 хвилин зашифровані VISIA, ПК лазерів, всі робочі файли. Клініка не може працювати поки не заплатить або відновить бекап.
🔌
MAC-адреса та OUI
▼
⚙ Технічно
MAC (Media Access Control) — унікальний 48-бітний ідентифікатор мережевої карти. Перші 3 байти (OUI) ідентифікують виробника. Nmap автоматично визначає виробника. «Intel Corporate» OUI = вбудований ПК (не телефон і не ноутбук) — так ідентифіковано медичні апарати.
💼 Для власниці
Як VIN-код автомобіля — перші цифри кажуть хто виробник. Ми побачили «Intel Corporate» і зрозуміли: це не телефон лікаря, це вбудований комп'ютер медичного апарату. Так вдалось ідентифікувати лазери у мережі навіть без назв пристроїв.
↔️
Lateral Movement — бокове переміщення між пристроями
▼
⚙ Технічно
Техніка пост-компрометації: зловмисник, зламавши один пристрій, використовує його як плацдарм для атаки на інші в тій самій мережі. В flat network без VLAN — будь-який скомпрометований хост дозволяє атакувати решту. Типові вектори: pass-the-hash, SMB relay, RDP, SSH.
💼 Для власниці
Хакер зламав камеру → з камери «перестрибнув» на ноутбук персоналу → з ноутбука → на сервер лазера. VLAN — це стіни що не дозволяють цьому «перестрибуванню» відбутись. Без стін — зламавши один слабкий пристрій, хакер отримує доступ до всього.
☠️
ARP Poisoning — отруєння таблиць маршрутизації
▼
⚙ Технічно
ARP (Address Resolution Protocol) зв'язує IP з MAC. Атака: надіслати фальшиві ARP-відповіді → жертва записує MAC зловмисника як MAC шлюзу → весь трафік жертви проходить через зловмисника (MITM). Інструмент: Ettercap. Ефективний проти Telnet, FTP, HTTP.
💼 Для власниці
Зловмисник «підміняє адресу» роутера. Коли адмін вводить пароль для входу в MikroTik — всі дані спочатку йдуть через зловмисника. Той бачить логін і пароль у відкритому вигляді. Відбувається непомітно — ніяких попереджень на екрані.
🌐
DNS Hijacking — підміна DNS-відповідей
▼
⚙ Технічно
Після захоплення MikroTik (.1) змінюється DNS-сервер для всіх клієнтів → DNS-запити на bank.ua повертають IP зловмисника → відкривається фішингова копія сайту. Або: DNS-відповіді для корпоративної пошти → перехоплення листів. MikroTik .1 має відкритий порт 53.
💼 Для власниці
Лікар відкриває банк у браузері — але бачить точну копію, і вводить дані картки. Або адміністратор відкриває «корпоративну пошту» — теж копія. Захопивши MikroTik, зловмисник може підробити будь-який сайт для всіх хто підключений до Wi-Fi клініки.
🔧
Ettercap — інструмент MITM атак
▼
⚙ Технічно
Ettercap — open-source GUI/CLI інструмент для MITM. Виконує ARP poisoning, DNS spoofing, перехоплення паролів HTTP/FTP/Telnet/POP3. Запуск ARP poison: ettercap -T -M arp:remote /[target]/ /[gateway]/. Входить до Kali Linux за замовчуванням.
💼 Для власниці
Безкоштовна програма що доступна на Kali Linux (спеціальний дистрибутив для пентестерів). Дозволяє перехоплювати паролі що передаються без шифрування — зокрема Telnet-пароль від MikroTik. Завантажується за 5 хвилин, запускається однією командою.
📡
Wireshark — аналізатор мережевого трафіку
▼
⚙ Технічно
Wireshark — GUI packet analyzer що захоплює і декодує мережеві пакети в реальному часі. Після ARP poisoning (Ettercap) — весь трафік жертви видимий. Фільтр для Telnet: tcp.port == 23. Покаже логін і пароль як звичайний текст у полі Payload.
💼 Для власниці
Безкоштовна програма що «підслуховує» мережу. Якщо адміністратор підключається до MikroTik через Telnet — Wireshark покаже логін і пароль як простий читабельний текст. Саме через це Telnet (порт 23) — критична небезпека.

Вимкнути SMBv1 на медичних ПК (.22 та .29)
SMBv1 = EternalBlue = WannaCry. Потрібно вимкнути на обох ПК лазерів. Якщо це Windows: через PowerShell. Попередньо переконатись що апарат не потребує SMBv1 для роботи (перевірити з виробником Asclepion/Candela).

Обмежити доступ до порту 9090 на VISIA (192.168.50.24)
Порт 9090 (XMLRPC) відповідає без авторизації. Варіанти: 1) Firewall на MikroTik — дозволити 9090 тільки з конкретного IP адміністративного ПК. 2) Windows Firewall на самому пристрої VISIA. 3) Зв'язатись з Canfield для закриття порту.
/ip firewall filter add chain=forward dst-address=192.168.50.24 dst-port=9090 src-address=!192.168.50.X action=drop

Найдорожчий фаєрвол не захистить якщо адміністратор дасть пароль першому хто попросить. Тренінг: як розпізнати pretexting («я від виробника лазера»), vishing (телефонне шахрайство), phishing (фальшиві email). Чіткий протокол: будь-хто хто просить IT-доступ → письмовий запит → підпис керівника → верифікація.
